Overwrite program

Overwrite is a program that overwrites empty space on disk, data and metadata.

The empty space on disk can contain sensitive data and information that can be recovered with recovery software and forensic, this can be privacy and security risk.

The operating systems and programs automatically write and delete data on disk, for example text editors may save copies of the document for recovery and later delete them. Also when users delete files they are only marked as deleted but the files remain on disk and can be recovered.

To protect privacy and security the remaining data on empty space on disk has to be overwritten.

To properly clean remaining data both metadata as file names and content has to be overwritten, this program has parameters as -meta and -data that specify quantity of data and metadata has to be overwritten.


Features

  • It is safe, does not overwrite existing files, and use operating system for overwrite.
  • Can be used on different file systems and operating systems,
    as NTFS UFS EXT FAT Windows Linux Unix.
  • Different options can be specified, as block size, number of files, quantity of data.
  • It is free and open source with general public licence.

Download

Source code and executables for different architectures and operating systems can be found in code repositories.

Overwrite program is written in C programming language it is contained in one file, can be compiled on different operating systems and also can be used without compilation, the source code can be run directly with Tiny C compiler ex: tcc -run overwrite.c

Overwrite user interface overwrite-ui.exe is written in C# programming language, and also this program can be used in Windows and Linux. Requirements: .Net Framework 2.0 for Windows and mono-complete package for Linux.

Usage

Overwrite empty space on disk data and metadata.
Overwrite Version 1.6 2019-11-21

USAGE: 
overwrite [-h -v -test -one -rand -block:] (-meta: &| -data:) -path: 
  -h 		 Print help and usage message. 
  -v 		 Print program version. 
  -test 	 For test, don't delete written files. 
  -one 		 Overwrite with 1, default with 0. 
  -rand 	 Overwrite with random data, default with 0. 
  -block 	 Block size, default 4096 bytes as NTFS and EXT4. 
  -meta 	 Number of metadata to write, ex: 1, 10.
  -data 	 Quantity of data to write, 1mb, 1gb, all, default mb. 
  -path 	 Path to directory where to write. 

EXAMPLE: 
- Write 10 metadata and 10Mb data, on Windows 
overwrite -meta:10 -data:10 -path:c:\ 
- Write random, block 512 bytes, 10Mb data, on Linux 
overwrite -rand -block:512 -data:10 -path:/mnt/usbdisk/

Parameters
Test
This parameter is used for testing, to analyze written files.
One
The data will be overwritten with one, all bits set to one.
Rand
The data will be overwritten with random content, bits one and zero.
Block
Block size on every data write. Operating System write data in block/sector of bytes.
This is important for overwrite because entire blocks has to be ovewritten.
File systems may have different block size, NTFS and EXT (hard disk) have block 4096 bytes, FAT (USB Stick) has block 512 bytes. Disk sector , Block data storage
Meta
Number of metadata entries to overwrite.
If the metadata is not overwritten the forensic software will see that the files where existed, their name size date and so on.
Data
Quantity of data to write, this parameter will write one big file with data.
The content of written data is specified by parameters -one and -rand, the size is specified in mb(Megabytes), gb(Gigabytes), all, default is mb.
Path
Path where to write data and metadata.

How it works

Overwrite program write small files and one big file using the operating system, and in this way overwrite the remaining data and metadata on disk.

When operating system delete file the space used by it is marked as free for new files, but the data and metadata is not cleaned and remain on disk, when the new files are written the remaining data is overwritten.

The File System has metadata that is summary information about the files and then data, writing small files overwrite metadata, and writing big file overwrite data, this is what the overwrite program does.

Limitations

Unicode
Unicode path is not supported in command line overwrite.c on Windows.

Metadata
Overwrite of metadata is not always guaranteed, this depends on different factors as number of files written, file name length, file system.

On some file systems the metadata has to be overwritten in the same directory of deleted files, and -meta parameter should have bigger number than metadata entries to clean.

This behavior has been reported in testing on FAT, EXT2, EXT3, EXT4 file sytems.

To see the testing results visit the link:
Overwrite Testing

NTFS File System
Metadata should not be filled all at once , the parameter -meta should be for example 100, 1000, it is not good practice more than 10.000. This is not restriction just good practice, if more that 10.000 files has been deleted then -meta parameter can be bigger.

NTFS file system metadata structure grow for maximum number of files written at one point in time, when files are deleted metadata entries are marked as free and ready for new files but the space is not released.
